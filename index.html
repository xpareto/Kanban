1. Arhitectură generală

ProjectBoard este o aplicație single-file HTML/JS/CSS fără dependențe externe (cu excepția Firebase CDN și Google Calendar REST API). Rulează ca PWA instalată din Chrome pe Android.

2. Structura datelor (State S)

S = {
  items: Project[],          // Proiecte cu tasks
  freeTasks: FreeTask[],     // Tasks libere (NEXT + markeri zone)
  nextOrder: OrderEntry[],   // Ordinea NEXT (free + proj refs)
  waitOrder: OrderEntry[],   // Ordinea WAIT
  waitFreeTasks: FreeTask[], // Free tasks în WAIT
  shareTasks: FreeTask[],    // Tasks în SHARE inbox
  journal: JournalEntry[],   // Jurnal operații
  shareJournal: ...,
  mergeFlags: {},            // Selecții merge SHARE
  labels: string[],          // Etichete rapide
  // UI state
  view, editId, editTid, editWaitId, showAdd, ...
}

FreeTask

{
  id: string,           // 'id_' + random
  name: string,         // Textul taskului (prima linie) + URL-uri (linii 2+)
  note?: string,        // Notă privată
  ts: number,           // Timestamp creare
  modifiedTs: number,   // Timestamp ultima modificare conținut
  zone: 'green'|'red'|'yellow'|null,  // Zona de culoare în NEXT
  zoneRank: number,     // Poziția în cadrul zonei (0-based)
  zoneMts: number,      // Timestamp ultima modificare zonă (independent de modifiedTs)
  _deleted?: true       // Tombstone pentru sync ștergere
}

OrderEntry

// Free task:
{type: 'free', id: string}
// Project task:
{type: 'proj', projectId: string, taskId: string}

Project / Task

Project: {type:'project', id, name, tasks: Task[]}
Task: {id, name, status:'next'|'wait'|'done', ts, modifiedTs,
       zone?, zoneRank?, zoneMts?, note?}

4. Sistemul de zone colorate

Zonele sunt delimitate de perechi de marcatori (free tasks al căror name începe cu *1, *2, sau *3).

ZonăMarcatoriCuloareRol
Verde*3…*3#DCFCE7Prioritate absolută aleasă manual
Roșie*1…*1#FEE2E2Tasks cu termene — sortate cronologic
Galbenă*2…*2#FEF9C3Priorități secundare

detectAndSaveZones()

Parcurge S.nextOrder direct (nu getNR). Calculează zone și zoneRank pentru fiecare item. Marcatorii *1/*2/*3 primesc întotdeauna zone=null. Cal:: events sunt skipped. La fiecare schimbare: zoneMts=Date.now(), modifiedTs=max(modifiedTs+1, now). Salvează FK și SK.

zone+rank sync

La Push: detectAndSaveZones() se apelează automat. La Pull merge:

  • buildRemoteZoneMap(remoteFT, remoteItems) — construiește map id→{zone,rank,zoneMts} excluzând marcatorii și cal:: events
  • mergeNextOrderWithZones() — inserează items NOI în zona corectă după rank
  • repositionByZone() — mută items EXISTENTE cu zonă schimbată
  • insertByZoneRank() — inserare la poziția rank în zonă, recalculând markerii la fiecare apel

Conflict resolution zone (zoneMts)

mergeItemArray și mergeProjectTasks folosesc zoneMts ca arbitru independent pentru câmpurile zone, zoneRank, zoneMts — separat de modifiedTs care arbitrează conținutul taskului. Câștigă dispozitivul cu zoneMts mai mare.

6. Sincronizare Firebase

Push (fbPush)

Apelează detectAndSaveZones() înainte de payload. Trimite tombstones din pb-deleted-ids ca FreeTask-uri cu _deleted:true. Refuză push cu date demo.

Pull (fbPull) — flux merge

1. mergeProjects(local, remote)
2. mergeItemArray(freeTasks) — cu zoneMts arbitru pentru zone
3. Construiește knownIds din freeTasks + tasks
4. Filtrează remoteNew (absent din localOrdSet)
5. mergeNextOrderWithZones(localOrder, remoteNew, ...) — items noi
6. repositionByZone(order, ...) — items existente cu zonă schimbată
7. Elimină tombstone-uri din nextOrder
8. dedupNextOrder + dedupMarkers — safety net
9. traceLog('fbPull merge done')

Conflict resolution

CâmpArbitru
name, note, statusmodifiedTs (mai mare câștigă)
zone, zoneRank, zoneMtszoneMts (mai mare câștigă)
Ștergere itemTombstone _deleted:true cu modifiedTs

7. Google Calendar

Flux autentificare

La prima lansare din zi:
1. Silent re-auth (prompt:none) → dacă eșuează (interaction_required)
2. Login explicit (prompt:select_account)
3. OAuth OK → fetch calendarList → fetchCal pentru fiecare calendar
4. Filtrare events → calInjectEvents → showMorningModal

Filtrare events

Tip eventCondiție includere
All-day (start.date)evStart ≤ tomorrowF AND evEnd > todayF (end exclusivă: evEnd <= todayF = exclude)
Timed (start.dateTime)evStart ≤ tomorrowF AND evEnd < todayF = exclude (strict)
